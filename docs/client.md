# Клиентские модули

Клиент — нативные ES-модули в браузере (без сборщика), лежат в `public/js/`.
Точка входа подключается из `index.html` как `<script type="module">`.
Принципы — в [architecture.md](architecture.md).

## Точка входа и конфигурация

| Модуль | Назначение |
|---|---|
| `js/main.js` | Оркестратор: `render()`, привязка событий, сценарии входа/выхода, `init()`. При входе в комнату (`JOINED`) сразу прописывает `?room=КОД` в адресной строке, чтобы ссылку-приглашение можно было скопировать прямо из URL. Модалки (настройки, ник, выход) закрываются кликом по затемнённому фону. Если вход в комнату не удался (`ERROR` — например, сервер перезапущен и комнаты больше нет), клиент сбрасывает сохранённый код, останавливает авто-переподключение и возвращает игрока на главный экран. Текст ошибки показывается только при явной попытке входа (ввод кода или ссылка-приглашение); при тихом восстановлении сессии по сохранённому коду ошибка не показывается. |
| `js/config.js` | `DEFAULTS` (настройки по умолчанию) и `WORDS` (словарь без дубликатов). |

## Интерфейс (`js/ui/`)

| Модуль | Назначение |
|---|---|
| `ui/screens.js` | `show` — навигация между экранами. |
| `ui/board.view.js` | `renderBoard` — игровое поле (CSS Grid N×N с равными колонками `1fr` и рядами `grid-auto-rows:1fr`, центрировано, ограничено 910×600; все карты одинакового размера независимо от длины слова — раньше flex-перенос делал ряды разной высоты). Угадывающему невскрытые карты показываются нейтральным светлым фоном — цвет ему неизвестен до открытия. Клик по карте — **голос** агента (task 1): кружки проголосовавших (цвет по id, свой — в белой обводке) рисуются в углу карты, а при единогласии снизу появляется 2-сек лоадер перед открытием. При старте партии капитану его карты ненадолго подсвечиваются (`spotlight`). Капитану уже открытые карты приглушаются прозрачностью (`dim-guessed`, task 3, `opacity:0.2`) — цвет угаданной карты остаётся виден сквозь приглушение, чтобы внимание было на оставшихся. |
| `ui/teams.view.js` | `renderTeams` — списки команд (кружок-аватар стабильного цвета по id игрока слева, пометка «это вы» справа), счёт (скрыт в лобби), карточка «Наблюдатели:» в шапке и ссылки-действия «Стать капитаном» / «Войти в команду» (видны в лобби и на паузе; «Стать капитаном» скрыта, когда в команде уже есть капитан — один капитан на команду). Карточка команды, чей сейчас ход, получает **неоновую подсветку** (`active-turn`, task 4). Плюс подвал карточки: **ввод подсказки** капитаном текущей команды (одно поле «слово + цифра» без плейсхолдера, отправка по Enter или кнопке «+», валидация «только одно слово»; если цифру не ввели — `parseClue` возвращает `number: null`, и в истории она показывается как «?») и кнопка **«Пропустить ход»** как голосование (task 1): кружки проголосовавших за пропуск под кнопкой и 2-сек лоадер при единогласии; клик шлёт `endTurn` (голос, повторный — снимает). **История подсказок** (`#…-clue-history-wrap`) вынесена из подвала и прижата к низу карточки (`margin-top:auto` во flex-колонке `.team-block`); вместо заголовка — `.clue-history-line` во всю ширину карточки, под которой список «слово + число/?». `bindTeamActions` навешивает обработчики один раз (DOM подвала статичен). |
| `ui/status.view.js` | `renderStatus` — только нижняя фиксированная карточка таймера `#timer-bar` (верхняя текстовая «плашка» фазы удалена: чей ход — по неоновой подсветке карточки команды (task 4), победитель — по заливке цветом `ui/modals.renderWin`). Карточка таймера видна только в активной игре (приклеена к нижнему краю по центру): подпись «Лидер думает:» (фаза clue) / «Команда думает:» (фаза guess) + таймер + одна кнопка паузы-переключателя (PAUSE/RESUME, подпись «⏸ Пауза» ↔ «▶️ Возобновить», стиль `btn-ghost` единый). На паузе подпись и значение таймера не меняются — слева обычный текст «думает», справа оставшееся время. |
| `ui/controls.view.js` | `renderControls` — кнопки старт/новая игра/в лобби/перемешать. Живут в нижней карточке `#game-controls` того же вида, что таймер (общий класс `.bottom-bar`, фиксирована у нижнего края по центру); видна в лобби (перемешать/начать) и после игры (новая/в лобби), скрыта целиком, когда у игрока нет активных кнопок. |
| `ui/settings.view.js` | `renderSettings`, `readSettingsForm` — форма настроек; кнопка «Перезапустить игру» (только хост, шлёт `newGame`). |
| `ui/log.view.js` | `renderLog` — журнал событий. |
| `ui/modals.js` | `renderWin` — при завершении игры заливает экран цветом команды-победителя линейным градиентом **позади всего UI** (`#win-flood`, `z-index:-1` — поверх корневого фона `html`, под контентом `body`, не перекрывает элементы). Насыщенный цвет команды (`--win-color`: синие `#043a5d`, красные `#5d1410`) у её края экрана (`--win-dir`: синие `to right`, красные `to left`) затухает в прозрачность к другому концу страницы, сливаясь с фоном (task 2). Слой держится всю фазу `over` и скрывается при её смене; отдельной модалки победы нет. `copyFeedback`, `openNickModal`. |
| `ui/toast.js` | `showToast` — всплывающие уведомления. |
