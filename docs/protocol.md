# Контракт WebSocket-сообщений

Все сообщения — JSON с обязательным полем `type`. Типы вынесены в константы и
используются обеими сторонами без «магических строк»: сервер —
`src/shared/messages.js`, клиент — `public/js/net/messages.js` (зеркало). При
добавлении нового типа обновляйте оба списка и этот документ.

## Входящие: клиент → сервер (`IN`)

| Тип | Поля | Кто вправе | Что делает |
|---|---|---|---|
| `createRoom` | `playerId`, `nickname`, `settings` | любой | Создаёт комнату и входит в неё. |
| `joinRoom` | `code`, `playerId`, `nickname` | любой | Входит в существующую комнату. |
| `setTeamRole` | `team`, `role` | игрок (в лобби или на паузе) | Меняет команду/роль. Капитан в команде только один: запрос на `spymaster`, когда капитан уже есть, понижается до `operative`. |
| `changeNickname` | `nickname` | игрок | Меняет ник (обрезка до 20 символов). |
| `updateSettings` | `settings` | хост (лобби/после игры) | Применяет валидные настройки (отдельно, без старта). |
| `startGame` | `words`, `settings?` | хост | Начинает партию. Если приложены `settings` — сперва применяет их (валидация как у `updateSettings`), затем стартует уже по новым настройкам. |
| `newGame` | `words`, `settings?` | хост | Начинает новую партию (то же, что `startGame`, включая необязательные `settings`). Кнопка «Сохранить и начать игру» в модалке шлёт именно его с настройками формы — работает в любой фазе (даже во время партии: поле собирается заново). |
| `backToLobby` | — | хост | Возврат из завершённой игры в лобби. |
| `giveClue` | `word`, `number` | капитан текущей команды | Даёт подсказку: `word` — ровно одно слово (с пробелом отклоняется), `number` — необязательная цифра-ориентир 0–9; если её нет (или невалидна), хранится `null` и показывается как «?». Подсказка дописывается в историю команды. Переход в фазу угадывания. |
| `editClue` | `index`, `word`, `number` | капитан (своей команды, во время игры) | Редактирует уже данную СВОЕЙ командой подсказку по позиции `index` в `clueHistory[team]` (task 1: исправить опечатку/число). Валидация `word`/`number` — как в `giveClue`. Если правится текущая активная подсказка (последняя у ходящей команды в фазе `guess`) — обновляется и `clue`. Фазу/таймер не меняет. Доступно только в фазах `clue`/`guess`. |
| `guess` | `index` | агент текущей команды (фаза `guess`) | **Голос** за карту, а не мгновенное открытие (task 1). Повторный `guess` по той же карте снимает голос; голос за другую карту/пропуск его перемещает (выбор эксклюзивен). Когда за карту проголосовали ВСЕ подключённые агенты команды, сервер запускает 1.5-сек отсчёт и затем открывает её. Верная карта НЕ передаёт ход — команда голосует дальше (счётчика попыток нет). |
| `endTurn` | — | агент текущей команды (фаза `guess`) | **Голос** за пропуск хода (task 1). Повторный `endTurn` снимает голос. При единогласии агентов сервер через 1.5-сек отсчёт передаёт ход. Капитан не голосует. |
| `pause` | — | игрок (в активной фазе) | Ставит игру на паузу. |
| `resume` | — | игрок (на паузе) | Снимает паузу. |
| `shuffleTeams` | — | хост (лобби/после игры) | Случайно раскидывает команды. |
| `setHost` | `playerId` | лидер комнаты или /admin | Меню модерации (task 1): передаёт корону лидера указанному игроку. Бота лидером не делает. Доступно в любой фазе. |
| `moveObserver` | `playerId` | лидер комнаты или /admin | Меню модерации (task 1): переводит игрока в наблюдатели (`team:null`, роль `operative`); снимает его «зависший» голос. Любая фаза. |
| `toggleStop` | — | любой игрок | F9-«стоп-пауза» (task 3): переключает общий полноэкранный оверлей с картинкой. Пока включён — таймер хода заморожен. Снятие повторным F9 заново запускает таймер активной фазы. |
| `adminAddPlayer` | `team` | /admin | Админ-панель (task 2): добавляет бота за `red`/`blue` или в наблюдатели (`team:null`). Бот — фейковый игрок без сокета (`bot:true`), всегда агент; в кворуме голосования/лидерстве/очистке не учитывается. |
| `adminAddClue` | `team` | /admin | Админ-панель: дописывает подсказку в историю команды `red`/`blue` (кнопка «Добавить ответ красным/синим»). Слово/число генерируются автоматически; фазу, активную подсказку и таймер не трогает. |
| `adminWin` | `team` | /admin | Админ-панель (task 2): имитирует победу команды `red`/`blue` — переводит игру в фазу `over`. |
| `adminXray` | — | /admin | Админ-панель (task 2): переключает личный X-ray. С ним игрок видит все цвета карт (как капитан), даже будучи агентом; раскрытие применяется только в ЕГО снимке. |
| `leave` | — | игрок | Выход из комнаты (logout). |

> Поле `admin:true` в `createRoom`/`joinRoom` помечает игрока как зашедшего по
> ссылке `/admin` (см. ниже) и выдаёт ему права админ-панели. Флаг сохраняется на
> игроке между переподключениями (авто-reconnect шлёт обычный `joinRoom`).
>
> **Режим `/admin`.** Открытие страницы по пути `/admin` (например,
> `домен/admin/?room=JSXA`) включает админ-режим в клиенте: показывается кнопка
> «Админ меню» и меню модерации при наведении на игрока. HTTP-сервер отдаёт по
> префиксу `/admin` ту же статику, что и из корня (см. `transport/httpStatic.js`).

> Сервер не доверяет клиенту: фаза, права (`hostId`/`role`/`team`) и границы
> значений проверяются на сервере; невалидные вызовы молча игнорируются.

## Исходящие: сервер → клиент (`OUT`)

| Тип | Поля | Когда |
|---|---|---|
| `joined` | `code`, `playerId` | Подтверждение входа в комнату. |
| `error` | `message` | Ошибка (например, комната не найдена). |
| `state` | полный снимок (см. ниже) | После любого изменения и на каждый тик таймера. |

### Снимок `state`

`code`, `hostId`, `you`, `players[]` (`id`, `nickname`, `team` — `red`/`blue`
или `null` для наблюдателя без команды, `role`, `connected`), `settings`,
`phase` (`lobby`/`clue`/`guess`/`over`), `board[]`
(`word`, `revealed`, `color`), `startingTeam`, `currentTeam`, `clue`
(`word`, `number` — `number` может быть `null`, тогда отрисовывается «?»),
`clueHistory` (`{red:[{word,number}], blue:[...]}` — все подсказки команды за
партию, для истории в карточке; `number` тоже может быть `null`), `votes`
(`{cards:{[index]:[playerId,...]}, skip:[playerId,...]}` — текущие голоса агентов,
task 1), `pendingVote` (`{kind:'guess'|'skip', index}` или `null` — идущий 2-сек
отсчёт единогласия), `timer`, `paused`, `stopped` (включена ли F9-«стоп-пауза» —
по нему рисуется оверлей с картинкой), `xray` (личный флаг X-ray текущего игрока
— по нему клиент знает состояние кнопки; чужие цвета в `board` уже раскрыты, если
он включён), `winner`, `remaining` (`red`, `blue`), `log[]`.

> `votes`/`pendingVote` видят все (там только id игроков — секретов не несут):
> клиент рисует по ним кружки проголосовавших и лоадер единогласия. Длительность
> отсчёта (1.5 с) задана на сервере (`gameEngine.VOTE_COUNTDOWN_MS`) и синхронна с
> CSS-анимацией лоадера.

> Счётчик попыток (`guessesRemaining`) удалён: верный ответ больше не приближает
> авто-передачу хода — команда сама решает, когда остановиться (см. правила ниже).

**Скрытие данных:** в `board[].color` цвет невскрытой карты приходит как `null`
обычному агенту; капитан, фаза `over` и агент с включённым X-ray (task 2)
показывают все цвета. См. сериализатор в [server.md](server.md) и
[architecture.md](architecture.md).
