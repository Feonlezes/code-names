# Запуск и доступ для других игроков

Как поднять сервер и дать друзьям зайти в комнату — в локальной сети и через
интернет. Сама игра authoritative-серверная: всё состояние комнаты живёт в
запущенном `node`-процессе, поэтому **сервер должен работать всё время, пока
идёт игра** (остановили — комната пропала).

## 1. Локально (на своей машине)

```bash
npm install
npm start            # http://localhost:3000  (порт меняется через PORT)
```

`localhost` доступен только на этом же компьютере — для друзей нужен один из
вариантов ниже.

## 2. Друг в той же сети (Wi-Fi/LAN)

1. Узнай свой локальный IP (адаптер вида `192.168.x.x`):

   ```powershell
   Get-NetIPAddress -AddressFamily IPv4 |
     Where-Object { $_.IPAddress -like '192.168.*' } |
     Select-Object IPAddress, InterfaceAlias
   ```

2. Открой порт `3000` в брандмауэре (один раз, **PowerShell от администратора**).
   `-Profile Any` важен, если сеть отмечена как «Общедоступная» (Public):

   ```powershell
   New-NetFirewallRule -DisplayName "Codenames 3000" -Direction Inbound `
     -Action Allow -Protocol TCP -LocalPort 3000 -Profile Any
   ```

3. Дай другу ссылку с твоим IP и кодом комнаты (4 символа из шапки «Комната»):

   ```
   http://192.168.0.135:3000/?room=КОД
   ```

Проверка: открой этот же адрес с телефона по той же Wi-Fi (мобильный интернет
выключи). Не открывается — почти всегда дело в брандмауэре (шаг 2).

## 3. Друг в другой сети (через интернет) — Cloudflare Tunnel

Локальный IP снаружи не виден; нужен туннель — он даёт временный публичный
`https://…`-адрес и пробрасывает его на твой `localhost:3000`. Открывать порты и
менять брандмауэр не нужно (соединение исходящее).

**Установка (один раз):**

```powershell
winget install --id Cloudflare.cloudflared
```

**Запуск туннеля** (в отдельном окне; `npm start` при этом работает в другом):

```powershell
cloudflared tunnel --url http://localhost:3000
```

> Отдельная пошаговая инструкция «создать туннель и узнать его ссылку» (в т. ч.
> надёжный способ получить адрес через файл) — в
> [cloudflare-tunnel.md](cloudflare-tunnel.md).

В первых строках вывода появится рамка с адресом:

```
+--------------------------------------------------------------+
|  Your quick Tunnel has been created! Visit it at:            |
|  https://случайные-слова-1234.trycloudflare.com              |
+--------------------------------------------------------------+
```

Собери ссылку для друга: этот адрес **+** `/?room=КОД`:

```
https://случайные-слова-1234.trycloudflare.com/?room=КОД
```

Игра по `https` работает: клиент сам поднимает защищённый `wss://`
(см. `public/js/net/socket.js` — протокол берётся из `location.protocol`).

**Важно:**

- Нужны **два окна одновременно**: `npm start` и `cloudflared`. Закрыл туннель —
  ссылка умерла; запустил заново — адрес `trycloudflare` стал другим (на
  бесплатном режиме он временный).
- Ссылку другу вставлять **в адресную строку браузера**, а не в поиск.

**Быстрая альтернатива без установки** — `localhost.run` по SSH (см. раздел 4).

## 4. Друг в другой сети — localhost.run (без установки, скрипт)

Если ставить `cloudflared` не хочется: в Windows 10/11 уже есть штатный `ssh`, а
сервис `localhost.run` поднимает туннель прямо по нему и выдаёт временный
публичный `https://…lhr.life`. WebSocket (`wss`), нужный игре, поддерживается.

**Вручную одной командой** (в отдельном окне, при работающем `npm start`):

```powershell
ssh -R 80:localhost:3000 nokey@localhost.run
```

В выводе будет строка с адресом, например
`https://случайные-символы.lhr.life`. Ссылка другу — адрес `+` `/?room=КОД`.

**Через скрипт `scripts/tunnel.ps1`** — он сам поднимает туннель, вытаскивает
адрес из вывода и печатает готовую (кликабельную в Windows Terminal / VS Code)
ссылку:

```powershell
npm run tunnel
# или сразу с кодом комнаты:
npm run tunnel -- -Room ABCD
# другой порт, если сервер не на 3000:
npm run tunnel -- -Port 4000
```

Скрипт выведет:

```
  Домен туннеля: https://случайные-символы.lhr.life
  Ссылка другу:  https://случайные-символы.lhr.life/?room=КОД
  Открыть/скопировать: https://случайные-символы.lhr.life   ← кликабельно
```

**Важно:**

- Окно со скриптом/`ssh` должно оставаться открытым — закрыл его, туннель умер.
  Остановить: `Ctrl+C` в этом окне (скрипт сам погасит `ssh`).
- Адрес `*.lhr.life` **временный**: при перезапуске туннеля он новый — отправь
  другу обновлённую ссылку.
- Если скрипт ругается «на порту 3000 никто не слушает» — сначала подними сервер
  (`npm start`) в другом окне.
- Нет `ssh`? Поставь компонент **OpenSSH Client**: Параметры → Приложения →
  Дополнительные компоненты → Добавить → OpenSSH Client.
